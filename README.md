## Engineering Identity

Works across the full lifecycle of AI systems — from evaluating whether a use case justifies investment, to specifying architecture, to overseeing implementation and holding the result to an evaluation discipline. The focus is governance that survives contact with real delivery: decisions are made in code as much as in policy documents.

Delivery is grounded in hexagonal architecture and strict typing on the backend (FastAPI, Pydantic V2, Python 3.12), paired with a modern typed frontend (Next.js 15 App Router, React, TypeScript, Tailwind CSS). AI features are built on Azure OpenAI's EU Data Zone with retrieval-augmented generation (ChromaDB, sentence-transformers), and shipped with the same rigor applied to the rest of the system — linting, static typing, structured logging, and automated testing as defaults, not afterthoughts.

## Featured Projects

<details>
<summary><b>AECT — AI Efficiency Control Tower</b></summary>
<br/>

AI use-case intake and governance advisory system for evaluating, prioritizing, and overseeing AI initiatives.

**Problem solved:** Organizations adopting AI need a consistent way to intake use-case proposals, evaluate ROI, and maintain governance oversight as initiatives move from proposal to production. AECT provides that structure as software rather than as a spreadsheet or a policy document.

**Architecture:** Hexagonal architecture separating domain logic from infrastructure, with a FastAPI backend exposing typed contracts (Pydantic V2) and a Next.js 15 App Router frontend consuming them.

**Stack:** FastAPI, Pydantic V2, Python 3.12, Next.js 15, React, TypeScript, Tailwind CSS, SQLite.

**Engineering depth:** Strict typing enforced end-to-end (mypy strict), linted with ruff, tested with pytest and Playwright, structured logging via structlog. Dependency and environment management handled with uv and pre-commit.

**Status:** In active development.
